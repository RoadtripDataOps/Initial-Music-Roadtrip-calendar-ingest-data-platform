# Source Trust And Partner Reporting

Milestone 5.1 adds source trust scoring and partner/destination reporting for
Music Roadtrip's internal ingestion platform.

The goal is to help the team see which sources, vendors, regions, and partner
submissions are producing clean, app-ready Concert event data. Scores and
reports are advisory. They do not auto-publish, auto-crawl, bypass admin
review, or make live provider calls.

## Source Types

`source_quality_scores` can score:

- `master_calendar_source`
- `api_provider`
- `api_feed_run`
- `crawl_run`
- `import_batch`
- `destination_partner`
- `region`
- `unknown`

The current POC computes master calendar source, API provider, region, and
destination partner scores. Other source kinds are reserved for future rollups
and reporting detail.

## Metrics

Each score stores a point-in-time local snapshot:

- event count
- events created and updated
- duplicate candidate count and duplicate rate
- rejected count
- extraction successes, failures, and unsupported source count
- missing and bad ticket counts
- missing image count
- generic image blocked count
- selected image pending approval count
- missing venue count
- missing geo count
- app-feed ready count
- app-feed blocked count
- manual correction count
- last success and failure timestamps
- score inputs JSON
- recommendations JSON

Optional cache fields on `master_calendar_sources` store the latest
`source_trust_score`, `source_trust_grade`, and `last_quality_score_id` for
quick admin review.

## Scoring Model

The scoring model starts at 100.

Penalties include:

- extraction failures
- unsupported crawls
- duplicate candidates
- rejected or quarantined review output
- missing tickets
- bad or generic ticket links
- missing images
- generic provider images blocked by Image QA
- selected images still pending approval
- missing venue linkage
- missing geo
- app-feed blockers
- manual correction rate
- stale/no recent success

Positive signals include:

- app-feed-ready records
- successful extraction history
- good ticket coverage
- good image coverage
- trusted/approved source status
- multiple source claims supporting records
- recent success

Grades:

- `excellent`: 90-100
- `good`: 75-89
- `fair`: 60-74
- `poor`: 40-59
- `blocked`: 0-39
- `unknown`: missing or intentionally unscored data

Recommendations are deterministic and focus on the highest-impact issues first:
extraction failures, unsupported formats, duplicate rates, missing/bad tickets,
missing/generic images, missing venues, and missing geo.

Region/destination reports include:

- region name

- current reporting period
- event count
- app-feed-ready events
- POI count
- app-feed-ready POIs
- approved, pending, failed, and unsupported calendar sources
- events created and updated
- duplicate candidates avoided
- source claims created
- image issues

- generic images blocked
- missing ticket links
- ticket-link issues
- extraction failures
- source quality grades
- top performing sources

Reports can be viewed at `/admin/partner-reports/{id}` and exported as JSON or
CSV.

## Admin Workflow

1. Open `/admin/source-quality`.
2. Click `Compute All Scores` or queue a rollup job.
3. Review poor/blocked sources and recommendations.
4. Open `/admin/regions/{id}/report`.
5. Generate a destination report.
6. Download JSON or CSV for partner review.

Region quality pages also show source trust tables and the latest partner
report link.

## Jobs And Scheduling

Background job types:

- `source_quality_rollup`
- `all_source_quality_rollup`
- `region_partner_report`

Scheduled task families:

- `source_quality_rollup`
- `partner_report_export`

These jobs read local database rows only. They do not make live provider calls,
do not fetch source URLs, do not scrape CitySpark, do not scrape social
platforms, and do not add API keys.

## Future Automation Use

Source trust scores can eventually help decide:

- which sources should be crawled more often
- which sources should require manual review
- which partner regions are ready for launch
- which vendor feeds need contract, data-quality, or mapping follow-up
- which calendars need outreach for better ticket links, venue data, or images

For now, trust scores do not change crawl gates, approval gates, dedupe rules,
ticket QA, image QA, app-feed privacy, or publication status.

Concert records remain `category=Concert` and `record_type=event`. Concert
records are events, never POIs. Category values other than Concert remain
POI/place style records.
